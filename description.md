# 一、 shoe factory C端 概述
```
   客户并非只想做一个普通鞋类电商网站，而是希望打造一个以“工厂直连消费者+个性化定制”为差异化的DTC品牌平台，面向海外市场。其本质是：利用真实生产资源建立信任壁垒，通过定制化提升客单价与粘性，并为未来平台（设计师生态）预留想象空间。 
```

# 二、 shoe factory 用户需求

## 2.1. “建立面向海外终端消费者的品牌商城，形成成品售卖、定制下单和品牌承接能力” 
```
表面需求	
    做一个海外版品牌官网	
真实需求
    ① 具备独立站级别的DTC能力，摆脱对亚马逊等平台的依赖；
    ② 品牌形象要专业、可信、本土化（海外审美、支付、物流、客服）；
    ③ 核心转化路径必须同时支持“买成品”和“定制下单”，两种模式互不干扰；
    ④ 商城本身就是品牌载体，要能讲好“工厂故事”和“定制故事”。
隐性痛点：客户可能已有供应链或成品线，但缺乏直面海外消费者的渠道和品牌沉淀能力。需要一个“既能卖货又能传递品牌价值”的一体化站点。
```

## 2.2. “建立固定鞋型下的用户上传图片定制能力，支持效果预览并直接付款下单”
```
表面需求	
    实现图片上传+预览+支付	
真实需求
    ① 定制门槛要极低：用户不需要懂设计，只需上传图片即可完成个性化；
    ② 预览效果必须接近真实（材质、颜色、图案贴合），否则退货率高；
    ③ 支付流程要无缝衔接定制环节，避免传统“先选款后定制”的割裂感；
    ④ 固定鞋型是刻意限制：不是技术做不到自由定制，而是为了生产标准化、降低成本、控制质量。
隐性痛点：客户不希望做“全参数可选”的复杂定制系统（开发与生产风险高），而是用“固定鞋型+用户图片”这种低成本、易上手的方式快速验证市场。预览的真实性是转化核心。
```

## 2.3. “建立直播入口和基础订单进度展示能力，强化真实工厂和真实制作的品牌差异化”

```
表面需求	
    加直播按钮+显示订单状态	

真实需求
    ① 要解决海外消费者对“进口/小品牌”的不信任：通过直播展示工厂实况，证明不是小作坊；
    ② 将“生产透明度”作为核心卖点：我卖的不只是鞋，而是“你能看着做的鞋”；
    ③ 降低售后服务压力：用户看到订单从“布料裁剪”到“缝合”的过程，会减少催单和疑虑；
    ④ 直播不是为了带货，而是为了品牌背书：可能只是定点工厂直播流，不一定需要互动。
隐性痛点：海外消费者对中国生产的品质和真实性常有顾虑。客户想用“真实镜头+进度可视化”取代华而不实的宣传语，建立信任型差异化。
```

## 2.4 “为未来素材商城、设计师入驻和设计佣金机制预留扩展能力”
```
表面需求	
    代码/架构上留好接口
真实需求
    ① 客户的长远商业模式是“平台化”：从自营定制走向C2M（消费者到制造商）+ 设计众创；
    ② 不想二次重构：当前版本的数据库、用户体系、订单分佣逻辑必须能平滑升级；
    ③ 设计师入驻意味着需要：设计师作品库、分润计算、作品使用授权、争议处理机制等；
    ④ 素材商城则要求：元数据管理（图片/图案可被搜索、付费授权、应用到定制）。
隐性痛点：客户可能已有设计师社区资源或计划引入海外本地设计师，但现阶段先跑通“自营定制+成品”闭环。如果现在就要求完整平台，成本高、风险大，所以强调“预留扩展”。这是判断客户成熟度的重要信号：他不是首次创业的小白。
```
